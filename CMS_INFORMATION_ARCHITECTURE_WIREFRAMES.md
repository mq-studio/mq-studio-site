## Information Architecture Overview

### Principles

The CMS architecture prioritizes:
- **Non-technical usability** - Moura requires no Markdown/code knowledge
- **Content-centric design** - Content creation/editing is the primary workflow
- **Intuitive hierarchy** - Clear mental model of site sections
- **Minimal cognitive load** - Obvious actions and clear affordances
- **Safety** - Data loss prevention, easy recovery, confirmation dialogs

### User Personas

**Primary User: Moura Quayle**
- Non-technical content creator
- Works primarily on desktop (occasional iPad)
- Needs to manage 4 content types: Musings, Artworks, Publications, Projects
- Wants quick content creation (5-10 minutes per post)
- Requires visual feedback and confirmation before publishing

**Secondary User: David Fushey**
- Provides notes/context for musings
- Reads published content
- Occasional bulk operations

### Navigation Pattern Rationale

- **Dashboard First** - Users enter at dashboard to see overview
- **Content Grouped** - All content management under "Content" section
- **Media Separate** - Media library is standalone for bulk operations
- **Taxonomy Grouped** - Tags/categories together for consistent management
- **Settings Last** - Least frequently accessed, doesn't clutter main nav

## Responsive Design Considerations

### Breakpoints

```
Mobile:   < 640px   (Portrait phones)
Tablet:   640px-1024px (iPad, landscape phones)
Desktop:  > 1024px  (Laptops, desktops)
```

### Mobile-Specific Changes

1. **Navigation** - Hamburger menu (drawer)
2. **Editor** - Full-screen focus mode
3. **Lists** - Single column, simplified view
4. **Media** - Single column grid with vertical thumbnail
5. **Modals** - Full-screen instead of centered

### Touch Targets

- Minimum 44px x 44px for all interactive elements
- Spacing between targets: minimum 8px
- Large buttons for primary actions
- Long-press context menus instead of hover menus

### Performance Considerations for Mobile

- Lazy load images in media library
- Simplified toolbar in editor (most-used options)
- Virtual scrolling for long lists
- Debounced search/filter
- Optimized image uploads (auto-resize)

### Keyboard Navigation

- All interactive elements accessible via Tab
- Logical tab order (top to bottom, left to right)
- Escape closes modals and overlays
- Arrow keys navigate lists/menus
- Enter/Space activates buttons

### Screen Reader Support

- Semantic HTML (buttons, labels, landmarks)
- ARIA labels for icon-only buttons
- Form labels associated with inputs
- Status messages announced (auto-save, errors)

### Visual Design

- Minimum contrast ratio 4.5:1 for text
- Focus indicators clearly visible (outline or highlight)
- Color not sole means of conveying information
- Error messages in text, not just color
- Alternative text for all images
