# Brand Constraints - Moura Quayle Living Archive

## Established Brand Elements

### Approved Color Palette
From the project control document:

**Primary Colors**:
- Primary Text: Charcoal (#333333)
- Background: Paper White (#f8f8f4)
- Primary Accent: Bright Teal (#009698)
- Secondary Accent: Vibrant Magenta (#D33479)
- Subtle Divider: Stone Gray (#cccccc)

**Spot Colors** (from artwork):
- Deep Purple (#7D2846)
- Additional colors from paintings as needed

### Typography Requirements
**Approved Typefaces**:
- Headlines & Titles: Montserrat (geometric sans-serif)
- Body Copy: Lora (readable serif)
- Both optimized for screen reading

### Existing Visual Assets
- High-resolution scans of watercolor paintings
- Shufa (calligraphy) artwork
- Natural-light portraits of Moura
- Cold-press watercolor paper texture

## Technical Constraints

### Performance Requirements
- Initial load time: <3 seconds
- Responsive breakpoints: Mobile (320px), Tablet (768px), Desktop (1024px+)
- Image optimization: WebP with JPEG fallback
- Maximum page weight: 2MB (excluding images)

### Browser Support
- Chrome/Edge: Last 2 versions
- Firefox: Last 2 versions
- Safari: Last 2 versions
- Mobile browsers: iOS Safari 14+, Chrome Android

### Accessibility Requirements
- WCAG 2.1 AA compliance minimum
- Keyboard navigation for all interactive elements
- Screen reader optimized content structure
- Color contrast ratios: 4.5:1 minimum, 7:1 preferred

## Content Constraints

### Archive Requirements
- Must handle diverse content types (PDFs, images, text)
- Search functionality across all content
- Clear attribution for David Fusthey's work
- Download capabilities for academic use

### Musing Requirements
- Support for text and audio formats
- Clear distinction from archived content
- RSS/subscription capability
- Comment/engagement features (if desired)

### Gallery Requirements
- Two distinct sections: Watercolor/Painting and Shufa
- High-quality image presentation
- Zoom/detail view capabilities
- Artwork metadata display

## Legal/Compliance Constraints

### Copyright Considerations
- Clear licensing information for downloads
- Attribution requirements
- Academic fair use guidelines
- Image protection for artwork

### Privacy Requirements
- GDPR compliance for any user data
- Contact form data handling
- Analytics consent (if used)
- No tracking without permission

## Budget/Resource Constraints

### Development Scope
- 2-week timeline (firm)
- Existing team of 4 specialists
- Leverage AI assistance for efficiency
- Focus on MVP features first

### Maintenance Considerations
- Low-maintenance architecture required
- Self-service content updates
- Minimal ongoing technical support
- Future-proof technology choices

## Non-Negotiable Elements

1. **"Bright Academia" Aesthetic**: Must maintain throughout
2. **Living Archive Concept**: Not a static repository
3. **Integrated Narrative**: Moura and David's work together
4. **Quality Standards**: No compromise on accessibility or performance
5. **User Experience**: Researcher-friendly interface

## Flexibility Areas

1. **Interactive Features**: Can be phased
2. **Advanced Search**: Basic first, enhance later
3. **Social Features**: Optional for MVP
4. **Analytics Depth**: Start simple
5. **Animation Complexity**: Performance first

## Stakeholder Constraints

### Must-Haves (from document)
- Living Archive as primary identity
- Moura's Musings section
- Distinct artwork galleries
- Clean search and filtering
- Direct PDF downloads

### Nice-to-Haves
- AI-powered semantic search
- Serendipity engine for connections
- Audio support for musings
- Advanced interactive features

## Risk Factors

### Technical Risks
- Large archive size may impact performance
- Search complexity across diverse content
- Mobile experience for detailed content

### Design Risks
- Balancing vibrancy with readability
- Maintaining consistency across content types
- Avoiding overwhelming users with choices

### Mitigation Strategies
- Progressive enhancement approach
- Performance budgets from start
- Regular user testing checkpoints
- Fallback designs for complex features
